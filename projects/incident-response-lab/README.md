# Incident Response Lab

## Overview

This project documents incident response practice using simulated or lab-based security scenarios.

## Objectives

- Follow a basic incident response workflow
- Record investigation notes and timelines
- Identify affected systems, evidence, and likely impact
- Write a clear incident report

## Lab Environment

- Operating systems: Linux and Windows lab systems
- Tools used: Logs, terminal commands, documentation templates
- Lab type: Simulated incident response

## Tools Used

- Linux terminal
- Windows event/log review tools as applicable
- Markdown documentation
- Report templates
